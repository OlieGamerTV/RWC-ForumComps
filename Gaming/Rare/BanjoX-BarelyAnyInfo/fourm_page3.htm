Oh hey, I worked with some prototypes/early builds before (mostly X360/PS3 ones).

I lack a bit of informations (like the console you're trying to run it on, or the state of the build [debug, release - if any is available], I only know you posess the *.xbe executable) so please excuse me if I post any outdated/old/obvious information, but I'll try to help as much as I can.

The build may be unbootable due to 3 facts (as I'm assuming that any code that was written was successfully compiled, as the *.xbe file is available):

* the game code's main header file has a bunch of #ifdef/#ifndef 's tokens, one of them could be something like _BOOTABLE / __runnable value that was either ignored or set to false. In that case you'd have to really fiddle around with the executable or do a reverse engineering (however it would take a whole lot of time and patience, plus I'm not really sure about the xbe's reverse engineering process) to recompile the code again under XDK libraries, using the correct tokens.

* the build was compiled to be ran only on a dev unit. Test/dev kits/units usually differ a bit from the normal consoles, as they have a whole bunch of extra features that are helping during game development or have different specs from the normal, end-user version (like for example Xbox's 360 dev unit has a 1024 and 512 MB RAM mode and you choose either depending on the memory usage and the optimalization of your code). So there's a glimmer of home the code/xbe may be executed, but only on the development console with all the libraries.

* it's a placeholder/dummy *.xbe executable or an executable filled with garbage data that was only used at one point as a container of some sort for the assets (i.e. to see how 3D models/sprites/textures would look on Xbox console)

Aha, I see! Thanks for the information Torrentstorm! Also thanks for the positive feedback mates. :)

And most likely you are correct, those are probably storage/container files for the textures/models. It's a common tactic among developers - within a first week or two they are getting familiar with the new equipment and machines - the four executables look like a release, debug, developer etc. builds (which makes a bit of sense since D(ebug) build is the largest) - most probably "Hello world" console version. The executables were packed with different assets just to see how would they look on the new Xbox console and also most likely to get familiar with new file formats, structures etc. (different assets were stored there, hinting that they tried to see/learn as much as possible). Most likely if booted somehow they would show different assets, moveable/scaleable/rotateable. By getting a TimeDate stamp from the executables we could tell more (if it dates from September/October 2002 then most likely they were just test/training builds).

However now I'm a bit more curious about the assets stored there. Both by the file formats (XBT? Raw data? Targas?) and about their overall existence. Either they were hinted to designers after Microsoft bought Rare, or they're even older (assuming that the design phase for Banjo-Threeie happened immediately after the release of Banjo-Tooie).
